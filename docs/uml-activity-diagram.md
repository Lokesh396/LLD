# Activity Diagram

An **activity diagram** is a behavioral UML diagram that models the workflow or flow of control of a system, showing sequential and parallel activities, decisions, and loops.

## Diagram

![Activity Diagram](uml/activityDiagram.drawio)

## Key Concepts

| Element | Meaning |
|---|---|
| Initial node (filled circle) | Start of the flow |
| Activity (rounded rectangle) | A step or action |
| Decision node (diamond) | Branching based on a condition |
| Merge node (diamond) | Merging conditional branches |
| Fork (thick horizontal bar) | Start of parallel flows |
| Join (thick horizontal bar) | Synchronize parallel flows |
| Final node (circle + filled circle) | End of the flow |
| Swimlane | Groups activities by responsible actor/component |
