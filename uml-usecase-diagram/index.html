    Use Case Diagram

    A use case diagram is a behavioral UML diagram that captures the functional requirements of a system by showing actors, use cases, and the relationships between them.

    Diagram

    Key Concepts

    Element Meaning
    Actor (stick figure) External entity that interacts with the system
    Use case (oval) A distinct functionality or goal
    System boundary (rectangle) Defines the scope of the system
    Association Actor participates in a use case
    <<include>> Use case always includes another use case
    <<extend>> Use case optionally extends another use case
    Generalization Actor or use case inherits from another
